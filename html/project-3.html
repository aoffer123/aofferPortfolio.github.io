New Leaf

Back to gallery

This application was created to be a simple, innovative and user-friendly solution for efficient nutrition tracking

Project Overview

Project Duration:

January - March 2022

The Problem:

Habits significantly impact our lives, influencing our thoughts, feelings, and actions. While cultivating good habits can lead to success and personal growth, eliminating bad habits remains challenging due to their immediate reinforcement. Most existing habit tracking apps focus on completion and data visualization but lack mechanisms to help users break bad habits and form good ones consistently.

Our Solution:

New Leaf addresses the issue of habit formation by encouraging users to replace bad habits with good ones through personalized reinforcement. The app allows users to create habits, categorize them, set short-term and long-term goals, and define rewards for achieving these goals. By providing notifications and progress tracking, New Leaf ensures users are consistently motivated to maintain their good habits.

Planning

  • Main goal identification & Implementation

    Our approach centers on user-centric design, starting with a homepage that lists all tracked habits. When creating a new habit, users enter a title, description, category, and both short-term and long-term goals, along with corresponding rewards. This structure sets New Leaf apart from other habit-tracking apps by emphasizing goal setting and rewards. Notifications alert users when they achieve their goals, prompting them to set new ones, and a detailed habit view allows users to track their progress and reward eligibility.

Design Process

  • Research & Wireframes

    The design process began with extensive research of existing applications in the market. This phase involved analyzing competitors to identify best practices, common features, and user pain points. By understanding what worked well and what didn't, I was able to gather valuable insights that informed our design decisions.

    • Following the research phase, I created paper wireframes to quickly visualize the layout and functionality of our application. These low-fidelity sketches allowed us to iterate rapidly and refine our ideas.

Design Process

  • High-fidelity Prototypes

    After finalizing the paper wireframes, I took on the responsibility of developing high-fidelity prototypes using advanced design tools. These prototypes included detailed visual design elements, interactions, and animations to closely mimic the final product. My focus was on ensuring the design was visually appealing and user-friendly. The prototypes were created using Figma, which allowed for precise control over the design elements and seamless collaboration with the development team.

    • In addition to visual elements, I incorporated interactive components to simulate user interactions and workflows. This allowed for thorough testing and validation of the user experience before moving into the development phase. The high-fidelity prototypes were presented to stakeholders for feedback and approval, ensuring that all design aspects met user needs and expectations. This process was crucial for identifying any potential issues early and making necessary adjustments to create a seamless and intuitive final product.

Development

  • Implementation

    As the lead front-end developer, I was responsible for translating the high-fidelity wireframes into functional code for each page of the application. Utilizing React JS and the MUI component library, I implemented a responsive and interactive user interface. This involved creating reusable components, managing state with React's hooks, and ensuring a seamless user experience across different devices and screen sizes. My focus was on maintaining clean, modular code and adhering to best practices in front-end development.

    • In addition to front-end implementation, I also set up the Firebase database to facilitate integration with the application. This included configuring Firebase Authentication for user management and Firestore for real-time data storage. My role involved writing the necessary backend code to connect the React components to the Firebase services, ensuring secure and efficient data handling. This setup allowed for real-time updates and a smooth user experience, critical for the functionality of our meal planner and diet tracker.
